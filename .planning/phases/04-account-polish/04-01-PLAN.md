<objective>
Implement customer order history (localStorage-based) and admin dashboard revenue card.

Purpose: ACCOUNT-01/02 require customers to view past orders — but backend has NO public order lookup endpoint (only admin endpoints exist). The pragmatic solution is to persist order data from checkout responses in a Zustand store with localStorage. DASH-02 requires a revenue figure — backend dashboard endpoint lacks revenue data, so we calculate it client-side by summing completed order amounts.

Output: MyOrdersPage showing recent orders, OrderConfirmationPage saving to history, DashboardPage with revenue card.
</objective>
