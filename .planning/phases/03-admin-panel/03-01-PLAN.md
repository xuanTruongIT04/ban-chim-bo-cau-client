    **src/hooks/admin/useAdminProducts.ts:**
    - `ADMIN_PRODUCTS_KEY = 'admin-products'` and `ADMIN_PRODUCT_KEY = 'admin-product'` as query key constants
    - `useAdminProducts(params: { page?: number; per_page?: number })` — useQuery with keepPreviousData

    - `useCreateProduct()` — useMutation calling adminProductApi.create, onSuccess invalidates ADMIN_PRODUCTS_KEY
    - `useUpdateProduct()` — useMutation calling adminProductApi.update, onSuccess invalidates both list and detail keys
    - `useDeleteProduct()` — useMutation calling adminProductApi.delete, onSuccess invalidates ADMIN_PRODUCTS_KEY

    - `useDeleteProductImage()` — useMutation for adminProductApi.deleteImage, invalidates detail key
    - `useAdjustStock()` — useMutation for adminProductApi.adjustStock, invalidates both list and detail

    All mutations use `useQueryClient()` for invalidation. Use `message` from antd for success/error toasts in onSuccess/onError callbacks.
