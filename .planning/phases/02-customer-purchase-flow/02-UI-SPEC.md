## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge offsets, chip inner padding |
| sm | 8px | Button inner gap, list item row gap, input prefix gap |
| md | 16px | Card body padding, form field vertical gap, page horizontal padding (mobile) |
| lg | 24px | Header paddingInline (already set), section gap, page horizontal padding (desktop) |
| xl | 32px | Product grid gap, page section vertical margin |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Header height (already set in CustomerLayout) |

Exceptions:
- Cart badge touch target: 44px minimum (mobile accessibility) — wrap `ShoppingCartOutlined` in a 44px tap zone
- Product image aspect ratio container: 200px height on cards (not a spacing value, a sizing constraint)
- Checkout summary panel: 280px fixed width on desktop (not a spacing value)

## Color

Source: `src/config/antd-theme.ts` + Ant Design 5 design token defaults.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff (colorBgContainer) | Page background, header background, card backgrounds, form surface |
| Secondary (30%) | #f5f5f5 (colorBgLayout, colorFillAlter) | Page layout background behind cards, checkout summary panel background, cart item row hover state |
| Accent (10%) | #1677ff (colorPrimary — AntD default blue) | See reserved list below |
| Destructive | #ff4d4f (colorError — AntD default red) | Remove item from cart confirmation, out-of-stock disabled label |

Accent (#1677ff) reserved for exactly these elements:
1. "Thêm vào giỏ" (Add to Cart) primary button
2. "Đặt hàng" (Place Order) primary button on checkout page
3. Pagination active page number
4. Category filter active/selected state
5. Cart badge count background
6. "Xem chi tiết" (View detail) link on product card
7. Price display on product card and product detail page (`color: colorPrimary`)

Destructive (#ff4d4f) reserved for exactly these elements:
1. "Xóa" (Remove) button in cart item row
2. Out-of-stock text tag ("Hết hàng")
3. Form validation error messages

## Interaction Contracts

### Product Listing Page

- Category filter change: fire new `useQuery` with updated `filter[category_id]`; paginate resets to page 1 on filter change
- Search input: client-side `.filter()` on current page results; 300ms debounce on input; does NOT trigger new API call
- Sort change: fire new `useQuery` with updated `sort` param; page resets to 1
- Pagination: fire new `useQuery` with updated `page`; scroll to top of product grid
- Add to cart from card: `POST /cart/items`; on success show `message.success("Đã thêm vào giỏ hàng")` (1.5s); cart badge count updates via TanStack Query invalidation

### Product Detail Page

- Image thumbnail click: swap primary displayed image (local state only, no API call)
- Quantity `InputNumber` change: update local state only; no API call until "Thêm vào giỏ hàng" is clicked
- Add to cart: `POST /cart/items` with quantity from InputNumber; on success `message.success("Đã thêm vào giỏ hàng")`; reset quantity to 1

### Cart Page

- Quantity `InputNumber` change: debounce 500ms then `PATCH /cart/items/:id`; optimistic update local display, revert on error
- Remove item: `DELETE /cart/items/:id` immediately; invalidate `['cart']` query on success
- "Tiến hành đặt hàng" button: navigate to `/checkout`; disabled if cart is empty

### Checkout Page

- Form submit: validate client-side first (AntD Form rules); on pass, fire `POST /checkout` with `Idempotency-Key: crypto.randomUUID()`; on success navigate to `/orders/confirm` with order data via router state
- Order data passed to confirmation page via `useLocation().state.order` — NOT via URL params (avoids exposing sensitive data)

### Cart Token Recovery Flow

- On any cart API call returning 404 CART_NOT_FOUND or 401 CART_TOKEN_REQUIRED: `POST /cart` → store new token → retry original operation → no visible interruption to user
