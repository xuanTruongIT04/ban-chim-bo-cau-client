## Spacing Scale

Declared values (must be multiples of 4). Mapped to Ant Design 5 token equivalents.

| Token | Value | AntD Token Equivalent | Usage |
|-------|-------|-----------------------|-------|
| xs | 4px | `paddingXXS` | Icon gaps, tag internal padding |
| sm | 8px | `paddingXS` | Compact element spacing, form item gap |
| md | 16px | `padding` | Default element spacing, card inner padding |
| lg | 24px | `paddingLG` | Section padding, form section gap |
| xl | 32px | `paddingXL` | Layout gaps, sidebar section breaks |
| 2xl | 48px | — | Major section breaks, login card vertical padding |
| 3xl | 64px | — | Page-level top/bottom padding (login page centering) |

Exceptions:
- Admin sidebar width: 240px (fixed, not from scale — AntD Sider standard)
- Top header height: 64px (AntD Layout.Header default)
- Admin login card width: 400px (fixed card width, centered on viewport)
- Touch targets for interactive elements: minimum 44px height (accessibility floor)

## Typography

Ant Design 5 base font size is 14px. The following roles are declared for Phase 1 surfaces only.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.57 (AntD default) | Form labels, nav items, table cells, general text |
| Label/Caption | 12px | 400 (regular) | 1.5 | Helper text below form fields, secondary metadata |
| Heading (card/section) | 20px | 600 (semibold) | 1.4 | Admin login card title, layout section headings |
| Display (page title) | 24px | 600 (semibold) | 1.33 | Reserved for page-level h1 (admin layout header area) |

Weights declared: 400 (regular) and 600 (semibold) only — two weights maximum.

Source: Ant Design 5 typography defaults; body 14px is AntD `fontSize` base token.

## Color

Using Ant Design 5 semantic tokens via `ConfigProvider` `theme.token`. No raw hex values are hardcoded — all colors reference AntD token names so the `ConfigProvider` theme override propagates correctly.

| Role | AntD Token | Approximate Value | Usage |
|------|-----------|-------------------|-------|
| Dominant (60%) | `colorBgLayout` | `#f5f5f5` (gray-2) | Page background — admin login page background, admin layout background |
| Secondary (30%) | `colorBgContainer` | `#ffffff` | Cards (login card), sidebar background, top header background, content area |
| Accent (10%) | `colorPrimary` | `#1677ff` (AntD blue) | Login button, active sidebar item indicator, focused form field border |
| Destructive | `colorError` | `#ff4d4f` | Error messages on login form, 401/5xx toast notifications |

Accent (`colorPrimary`) reserved for:
1. The "Đăng nhập" submit button (login page)
2. Active/selected sidebar navigation item highlight
3. Focused state outline on Email and Password input fields
4. Spinner/loading indicator during login API call

Accent is NOT used for: nav links in default state, card borders, layout dividers, labels.

Source: CONTEXT.md D-16 (gray background), D-21 (sidebar), Ant Design 5 default palette.

## Surface Specifications

### Admin Login Page (`/admin/login`)

- **Page background:** `colorBgLayout` (`#f5f5f5`) — full viewport, centered content
- **Card:** `colorBgContainer` (`#ffffff`), width 400px, `borderRadius` 8px, AntD `boxShadow` medium (card shadow token)
- **Card vertical padding:** 48px top, 48px bottom (2xl token)
- **Card horizontal padding:** 32px left, 32px right (xl token)
- **Logo/title area:** Project name "Bán Chim Bồ Câu" as text in Display size (24px, weight 600), centered, margin-bottom 32px
- **Email field:** AntD `Input` — full width, label "Email", placeholder "admin@example.com"
- **Password field:** AntD `Input.Password` (with show/hide toggle) — full width, label "Mật khẩu", placeholder "••••••••"
- **Submit button:** AntD `Button` type="primary", full width, height 40px (AntD controlHeight default), label "Đăng nhập"
- **Loading state:** Button `loading={true}` prop — shows AntD spinner inside button, button disabled. No separate overlay.
- **Form layout:** AntD `Form` with `layout="vertical"` (label above field)
- **Card centering:** CSS flexbox — `display: flex; align-items: center; justify-content: center; min-height: 100vh` on the page wrapper

### Admin Layout (sidebar + top header)

- **Sider:** AntD `Layout.Sider`, width 240px, `colorBgContainer` background, `collapsible={false}` in Phase 1
- **Sider logo area:** Height 64px, centered "Bán Chim Bồ Câu" text (14px, weight 600), `colorBgLayout` background
- **Sider nav items:** AntD `Menu` component, `mode="inline"`, items: Dashboard, Sản phẩm, Đơn hàng — each with AntD icon
  - Dashboard → `<DashboardOutlined />`
  - Sản phẩm → `<ShoppingOutlined />`
  - Đơn hàng → `<OrderedListOutlined />`
- **Top header:** AntD `Layout.Header`, height 64px, `colorBgContainer` background, right-aligned content: admin display name + `<LogoutButton>`
- **Logout element:** AntD `Button` type="text" with `<LogoutOutlined />` icon + text "Đăng xuất", right-aligned in header
- **Content area:** AntD `Layout.Content`, `colorBgLayout` background, padding 24px (lg token)

### Customer Layout (top navbar)

- **Navbar:** AntD `Layout.Header`, height 64px, `colorBgContainer` background, full-width, sticky top
- **Logo:** Left-aligned — text "Bán Chim Bồ Câu" (16px, weight 600) or image if available
- **Category nav links:** Center or left-after-logo — AntD `Menu` component, `mode="horizontal"`, items populated in Phase 2
- **Cart icon:** Right-aligned — `<ShoppingCartOutlined />` from `@ant-design/icons`, wrapped in AntD `Badge` (count from cart store), size 20px icon

Note: Customer layout is scaffolded in Phase 1 as a structural shell. Category items and cart badge count are wired in Phase 2.

### Placeholder Pages (scaffolded but empty in Phase 1)

All non-auth routes rendered in Phase 1 are placeholder pages. They display:
- Page title in Display typography (24px, weight 600)
- Body text "Tính năng này sẽ có trong phiên bản tiếp theo." (14px, weight 400)
- No action buttons

## Interaction States

### Login Form

| State | Visual |
|-------|--------|
| Default | Standard AntD Input border (`colorBorder`) |
| Focused | AntD primary color border (`colorPrimary`) + box shadow |
| Validation error | AntD `status="error"` on Form.Item — red border + error message below field |
| Submitting | Button `loading={true}`, all form fields `disabled={true}` |
| Success | Immediate redirect — no success toast on login page |
| API error | AntD `Alert` component with `type="error"` rendered above the submit button — inline within card, not a floating toast |

### Protected Route (AdminRoute)

| State | Behavior |
|-------|----------|
| `isInitializing: true` | Render full-screen centered AntD `Spin` (size="large") — prevents route flicker (CONTEXT.md D-30) |
| Token present, role is admin | Render `<Outlet />` |
| No token | `<Navigate to="/admin/login" replace />` |
| Token present, role is not admin | `<Navigate to="/admin/login" replace />` (role check via `user.role === 'admin'`) |

### Sidebar Navigation

| State | Visual |
|-------|--------|
| Default | AntD Menu default item style |
| Active/selected | `colorPrimary` left border indicator — AntD Menu handles via `selectedKeys` prop |
| Hover | AntD Menu default hover (`colorBgTextHover`) |

## App Initialization Sequence

This sequence is a visual contract — it determines what the user sees during the auth store hydration window.

1. App mounts → auth store sets `isInitializing: true`
2. If `localStorage` has `auth_token` → call `GET /api/me`
3. During `GET /api/me` call → **all routes render full-screen `<Spin size="large" tip="Đang tải..." />`** centered on `colorBgLayout` background
4. `GET /api/me` success → store `user`, set `isInitializing: false` → render correct route
5. `GET /api/me` 401 → clear token, set `isInitializing: false` → render correct route (login redirect triggers naturally)
6. No token in localStorage → set `isInitializing: false` immediately → render correct route

Source: CONTEXT.md D-29, D-30 (isInitializing flag pattern)

## Ant Design 5 ConfigProvider Token Overrides

The following `ConfigProvider` token overrides are declared for Phase 1. All other tokens use AntD defaults.

```ts
// src/configs/antd-theme.ts
export const antdTheme = {
  token: {
    // Typography
    fontSize: 14,
    fontSizeHeading1: 24,
    fontSizeHeading2: 20,

    // Border radius
    borderRadius: 8,
    borderRadiusLG: 8,

    // Primary color (AntD blue — keep default for v1)
    // colorPrimary: '#1677ff' — default, no override needed

    // Layout
    // colorBgLayout: '#f5f5f5' — default, no override needed
  },
  components: {
    Layout: {
      headerBg: '#ffffff',
      siderBg: '#ffffff',
    },
    Menu: {
      // Sidebar menu uses white background (matches Sider)
      itemBg: '#ffffff',
    },
  },
};
```

Executor: wrap the entire app in `<ConfigProvider theme={antdTheme}>` inside `main.tsx`.
